# UI/UX Benchmark — Must Be Better Than HCMC Metro

## Design goal

The product should feel like a premium youth-focused transit companion, not a static information portal. HCMC Metro is the functional floor; EcoTransit must be more guided, more alive, more social and more helpful.

## Benchmark-to-upgrade mapping

| HCMC Metro pattern | EcoTransit upgraded pattern |
|---|---|
| Simple navigation menu | App-like shell with sticky route CTA, bottom nav on mobile, command search |
| Static route lookup | Interactive map + bottom sheet + route cards + weather badges |
| Metro-only stations | Bus + metro + walking route timeline |
| Nearby place list | Map/list dual mode, richer cards, review snippets, guide chips, “near this station” context |
| News list | Editorial guide hub with video/article cards, weather tags, station tags |
| Offers | Gamified rewards wallet and voucher redemption |
| Map layers | Premium map drawer with layers, route preview, station details and place discovery |
| Basic share | Designed “time bill” card with privacy controls and social metadata |

## Visual quality requirements

1. Use large clean typography and strong hierarchy.
2. Avoid dashboard-like clutter on public pages.
3. Use map and route timeline as hero interactions, not secondary widgets.
4. Cards must have meaningful content density, not generic empty rectangles.
5. Use gradients/glass/blur sparingly and consistently.
6. Motion must support comprehension: drawer slide, route reveal, reward success, bill card generation.
7. Loading states must be branded skeletons, not plain spinners everywhere.
8. Empty/error states must explain next action.
