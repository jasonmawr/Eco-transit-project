# Mobile-First Interactions

## General

- Minimum tap target: 44x44px.
- Avoid dense desktop tables on mobile; use cards or horizontal-safe patterns.
- Bottom sheets are preferred for route results and decision details.
- Sticky primary actions only when they do not cover content.
- Forms must preserve draft state where long.

## Route search

- Use large origin/destination fields.
- Swap origin/destination button.
- Use current location only with clear permission prompt.
- Results should be thumb-scrollable.
- Map should not trap scroll.

## Ticket upload

- Camera/photo library flow should be obvious.
- Upload progress visible.
- Do not ask for too much metadata before upload.
- Explain pending/manual review in plain language.

## Bill share

- Preview card must fit mobile width.
- Copy link button must be one tap.
- Native share optional.
- Privacy level text must be understandable.

## Admin on mobile

Admin can be usable on tablet/desktop first, but must not hard-break on 390px width. Queue tables can collapse into cards.
