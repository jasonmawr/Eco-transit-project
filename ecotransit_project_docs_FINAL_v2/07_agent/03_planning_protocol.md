# Planning Protocol

## Plan requirements

A good plan has 3-7 steps, not vague intentions.

Include:

- Docs read.
- Impact analysis.
- Data/API/UI changes.
- Test strategy.
- Rollback considerations.

## Planning bans

- Do not say "I'll implement everything" with no file/module breakdown.
- Do not ignore known unknowns.
- Do not silently expand P0 into P2.
- Do not start coding critical changes without clarifying irreversible decisions.
