# Project Vision

> EcoTransit Project Docs v1.0 — generated for coding-agent handoff.
> Principle: production-grade, mobile-first, privacy-by-design, không xây trùng lõi vận hành của Go!Bus/HCMC Metro; EcoTransit là lớp trải nghiệm, nội dung, gamification và chia sẻ.


## Product identity

EcoTransit is a youth-oriented public-transit experience web app for Ho Chi Minh City. The core product is not the official bus/metro operation layer. It is a user experience layer that makes public transit feel easier, more social, more guided, and more rewarding.

## Target users

- Age: 18-25.
- Location: Inner Ho Chi Minh City, initially District 1 and the administrative center.
- Context: Students, young workers, interns, tourists/local explorers, and people who are curious about trying bus/metro but need confidence and motivation.

## Strategic positioning

EcoTransit should win by being the layer that official operator apps usually do not prioritize:

- Clear, friendly route experience.
- Weather-aware route preference.
- Station-area lifestyle discovery.
- Guide/review content near stations/stops.
- Quiz-based onboarding.
- Trip bill/card for sharing.
- Rewards through verified transit behavior.

## Product promise

"Đi công cộng dễ hơn, có gu hơn, và đáng chia sẻ hơn."

## Core success metrics

| Metric | Target interpretation |
|---|---|
| Route searches | Users trust EcoTransit as planning entry point |
| Ticket uploads | Users are willing to prove trips for rewards |
| Verify success rate | Ticket pipeline is useful and not too manual-heavy |
| Bill shares | Product has social spread |
| Review/guide submissions | UGC loop begins to compound |
| Voucher redeems | Rewards are meaningful |
| Repeat weekly users | Habit loop works |

## Principles

1. Experience over operator duplication.
2. Privacy before virality.
3. Ledger before points cache.
4. Moderation before public UGC.
5. Adapter before provider lock-in.
6. Mobile-first before desktop polish.
7. Tasteful UI before generic templates.
